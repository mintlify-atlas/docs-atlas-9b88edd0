---
title: Custom Endpoints
description: Configure custom AI endpoints and providers
---

LibreChat allows you to add custom AI endpoints beyond the built-in providers. Configure them in `librechat.yaml` under the `endpoints.custom` array.

## Basic Structure

Each custom endpoint requires a unique name, API key, base URL, and models configuration.

```yaml
endpoints:
  custom:
    - name: 'endpoint-name'
      apiKey: '${ENV_VARIABLE}'
      baseURL: 'https://api.provider.com/v1'
      models:
        default: ['model-1', 'model-2']
        fetch: false
```

## Configuration Fields

<ParamField path="name" type="string" required>
  Unique name for the endpoint (displayed in the UI)
</ParamField>

<ParamField path="apiKey" type="string" required>
  API key for authentication (use environment variables)
  
  ```yaml
  apiKey: '${GROQ_API_KEY}'
  ```
</ParamField>

<ParamField path="baseURL" type="string" required>
  Base URL for the API endpoint
  
  ```yaml
  baseURL: 'https://api.groq.com/openai/v1/'
  ```
</ParamField>

### Models Configuration

<ParamField path="models.default" type="array" required>
  List of default models to use (at least one required)
  
  ```yaml
  models:
    default:
      - 'llama3-70b-8192'
      - 'mixtral-8x7b-32768'
  ```
</ParamField>

<ParamField path="models.fetch" type="boolean" default="false">
  Fetch available models from API
  
  <Note>When true, queries the API `/models` endpoint</Note>
</ParamField>

### Optional Settings

<ParamField path="titleConvo" type="boolean" default="false">
  Enable automatic conversation title generation
</ParamField>

<ParamField path="titleModel" type="string" default="gpt-3.5-turbo">
  Model to use for generating titles
  
  ```yaml
  titleModel: 'mixtral-8x7b-32768'
  ```
</ParamField>

<ParamField path="titleMethod" type="string" default="completion">
  Method for title generation: `completion` or `functions`
</ParamField>

<ParamField path="summarize" type="boolean" default="false">
  Enable conversation summarization
</ParamField>

<ParamField path="summaryModel" type="string" default="gpt-3.5-turbo">
  Model to use for summarization
</ParamField>

<ParamField path="modelDisplayLabel" type="string" default="AI">
  Label displayed for AI messages
  
  ```yaml
  modelDisplayLabel: 'Groq'
  ```
</ParamField>

<ParamField path="iconURL" type="string">
  Custom icon URL for the endpoint
  
  ```yaml
  iconURL: https://example.com/icon.png
  ```
</ParamField>

### Headers Configuration

<ParamField path="headers" type="object">
  Custom headers to include with requests
  
  ```yaml
  headers:
    x-portkey-api-key: '${PORTKEY_API_KEY}'
    x-portkey-virtual-key: '${PORTKEY_OPENAI_VIRTUAL_KEY}'
  ```
  
  Special variables:
  - `{{LIBRECHAT_BODY_PARENTMESSAGEID}}` - Insert parent message ID
</ParamField>

### Request Parameters

<ParamField path="addParams" type="object">
  Add or override default request parameters
  
  ```yaml
  addParams:
    safe_prompt: true
    temperature: 0.7
  ```
</ParamField>

<ParamField path="dropParams" type="array">
  Remove default parameters from requests
  
  ```yaml
  dropParams: ['stop', 'user', 'frequency_penalty', 'presence_penalty']
  ```
  
  <Note>Required for some providers like Mistral to avoid 422 errors</Note>
</ParamField>

## Example: Groq

```yaml
endpoints:
  custom:
    - name: 'groq'
      apiKey: '${GROQ_API_KEY}'
      baseURL: 'https://api.groq.com/openai/v1/'
      models:
        default:
          - 'llama3-70b-8192'
          - 'llama3-8b-8192'
          - 'mixtral-8x7b-32768'
          - 'gemma-7b-it'
        fetch: false
      titleConvo: true
      titleModel: 'mixtral-8x7b-32768'
      modelDisplayLabel: 'groq'
```

## Example: Mistral AI

```yaml
endpoints:
  custom:
    - name: 'Mistral'
      apiKey: '${MISTRAL_API_KEY}'
      baseURL: 'https://api.mistral.ai/v1'
      models:
        default: ['mistral-tiny', 'mistral-small', 'mistral-medium']
        fetch: true
      titleConvo: true
      titleModel: 'mistral-tiny'
      modelDisplayLabel: 'Mistral'
      dropParams: ['stop', 'user', 'frequency_penalty', 'presence_penalty']
```

<Accordion title="Why drop parameters for Mistral?">
  Mistral AI's API doesn't support some OpenAI-standard parameters. Dropping them prevents 422 validation errors.
</Accordion>

## Example: OpenRouter

```yaml
endpoints:
  custom:
    - name: 'OpenRouter'
      apiKey: '${OPENROUTER_KEY}'
      baseURL: 'https://openrouter.ai/api/v1'
      headers:
        x-librechat-body-parentmessageid: '{{LIBRECHAT_BODY_PARENTMESSAGEID}}'
      models:
        default: ['meta-llama/llama-3-70b-instruct']
        fetch: true
      titleConvo: true
      titleModel: 'meta-llama/llama-3-70b-instruct'
      dropParams: ['stop']
      modelDisplayLabel: 'OpenRouter'
```

## Example: Portkey AI

```yaml
endpoints:
  custom:
    - name: 'Portkey'
      apiKey: 'dummy'
      baseURL: 'https://api.portkey.ai/v1'
      headers:
        x-portkey-api-key: '${PORTKEY_API_KEY}'
        x-portkey-virtual-key: '${PORTKEY_OPENAI_VIRTUAL_KEY}'
      models:
        default: ['gpt-4o-mini', 'gpt-4o', 'chatgpt-4o-latest']
        fetch: true
      titleConvo: true
      titleModel: 'current_model'
      summarize: false
      summaryModel: 'current_model'
      modelDisplayLabel: 'Portkey'
      iconURL: https://images.crunchbase.com/image/upload/c_pad,f_auto,q_auto:eco,dpr_1/rjqy7ghvjoiu4cd1xjbf
```

<Note>
  `current_model` uses the currently selected model for titles/summaries
</Note>

## Azure OpenAI

While Azure OpenAI has a dedicated endpoint, you can also configure it as a custom endpoint:

```yaml
endpoints:
  azureOpenAI:
    # Azure-specific configuration here
```

Refer to the [official Azure OpenAI documentation](https://www.librechat.ai/docs/configuration/librechat_yaml/ai_endpoints/azure) for detailed setup.

## Model Specifications

Create preset configurations that can be grouped in the UI:

```yaml
modelSpecs:
  list:
    # Nested under endpoint
    - name: "gpt-4o"
      label: "GPT-4 Optimized"
      description: "Most capable GPT-4 model"
      group: "openAI"
      preset:
        endpoint: "openAI"
        model: "gpt-4o"
    
    # Custom group with icon
    - name: "coding-assistant"
      label: "Coding Assistant"
      description: "Specialized for coding tasks"
      group: "my-assistants"
      groupIcon: "https://example.com/icons/assistants.png"
      preset:
        endpoint: "openAI"
        model: "gpt-4o"
        instructions: "You are an expert coding assistant..."
        temperature: 0.3
    
    # Standalone (no group)
    - name: "general-assistant"
      label: "General Assistant"
      description: "General purpose assistant"
      preset:
        endpoint: "openAI"
        model: "gpt-4o-mini"
```

<ParamField path="group" type="string">
  Organize specs in UI:
  - Match endpoint name to nest under that endpoint
  - Custom name creates separate collapsible section
  - Omit for standalone top-level item
</ParamField>

<ParamField path="groupIcon" type="string">
  Icon for custom groups (URL or built-in endpoint key)
  
  <Note>Only needs to be set on one spec per group</Note>
</ParamField>

## Environment Variables

Set the following environment variables for your custom endpoints:

```bash
# Groq
GROQ_API_KEY=your_groq_key

# Mistral AI
MISTRAL_API_KEY=your_mistral_key

# OpenRouter
OPENROUTER_KEY=your_openrouter_key

# Other providers
ANYSCALE_API_KEY=
COHERE_API_KEY=
DEEPSEEK_API_KEY=
FIREWORKS_API_KEY=
HUGGINGFACE_TOKEN=
PERPLEXITY_API_KEY=
TOGETHERAI_API_KEY=
```

## Troubleshooting

<Accordion title="422 Validation Errors">
  Some providers don't support all OpenAI parameters. Use `dropParams` to remove unsupported fields:
  
  ```yaml
  dropParams: ['stop', 'user', 'frequency_penalty', 'presence_penalty']
  ```
</Accordion>

<Accordion title="Models Not Appearing">
  1. Check `models.default` has at least one model
  2. Verify `models.fetch: true` if querying API
  3. Ensure API key is valid and has access
</Accordion>

<Accordion title="Custom Headers Not Working">
  Make sure headers are under the `headers` key and use proper environment variable syntax:
  
  ```yaml
  headers:
    x-api-key: '${MY_API_KEY}'
  ```
</Accordion>
