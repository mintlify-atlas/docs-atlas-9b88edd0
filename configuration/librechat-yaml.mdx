The `librechat.yaml` file is the primary configuration file for LibreChat. It allows you to customize endpoints, interface settings, file storage, and more.

## Configuration Version

<ParamField path="version" type="string" required>
  Configuration file version
  
  ```yaml
  version: 1.3.4
  ```
</ParamField>

## Cache Settings

<ParamField path="cache" type="boolean" default="false">
  Enable response caching
  
  ```yaml
  cache: true
  ```
</ParamField>

## Interface Configuration

Customize the LibreChat user interface with these settings.

```yaml
interface:
  customWelcome: 'Welcome to LibreChat! Enjoy your experience.'
  fileSearch: true
  endpointsMenu: true
  modelSelect: true
  parameters: true
  sidePanel: true
  presets: true
  bookmarks: true
  multiConvo: true
```

### Interface Options

<ParamField path="interface.customWelcome" type="string">
  Custom welcome message displayed to users
</ParamField>

<ParamField path="interface.fileSearch" type="boolean" default="true">
  Enable file search as a chat area selection
  
  <Note>This does not disable the Agents File Search Capability. See Agents endpoint configuration for that.</Note>
</ParamField>

<ParamField path="interface.endpointsMenu" type="boolean" default="true">
  Show endpoints selection menu
</ParamField>

<ParamField path="interface.modelSelect" type="boolean" default="true">
  Enable model selection dropdown
</ParamField>

<ParamField path="interface.parameters" type="boolean" default="true">
  Show model parameters controls
</ParamField>

<ParamField path="interface.sidePanel" type="boolean" default="true">
  Enable side panel
</ParamField>

## Registration Settings

```yaml
registration:
  socialLogins: ['github', 'google', 'discord', 'openid', 'facebook', 'apple', 'saml']
  allowedDomains:
    - "gmail.com"
    - "company.com"
```

<ParamField path="registration.socialLogins" type="array">
  List of enabled social login providers
  
  Available options: `github`, `google`, `discord`, `openid`, `facebook`, `apple`, `saml`
</ParamField>

<ParamField path="registration.allowedDomains" type="array">
  Email domains allowed for registration
  
  <Note>Leave empty to allow all domains</Note>
</ParamField>

## Balance & Transactions

```yaml
balance:
  enabled: false
  startBalance: 20000
  autoRefillEnabled: false
  refillIntervalValue: 30
  refillIntervalUnit: 'days'
  refillAmount: 10000
```

<ParamField path="balance.enabled" type="boolean" default="false">
  Enable token balance system
</ParamField>

<ParamField path="balance.startBalance" type="number" default="20000">
  Initial token balance for new users
</ParamField>

<ParamField path="balance.autoRefillEnabled" type="boolean" default="false">
  Enable automatic balance refill
</ParamField>

<ParamField path="balance.refillIntervalValue" type="number">
  Refill interval value (e.g., 30)
</ParamField>

<ParamField path="balance.refillIntervalUnit" type="string">
  Refill interval unit: `days`, `hours`, or `minutes`
</ParamField>

```yaml
transactions:
  enabled: false
```

<ParamField path="transactions.enabled" type="boolean" default="true">
  Save transaction records to database
  
  <Note>If balance.enabled is true, transactions are always enabled</Note>
</ParamField>

## Agent Actions Domain Restrictions

```yaml
actions:
  allowedDomains:
    - 'swapi.dev'
    - 'librechat.ai'
    - 'google.com'
    - 'http://10.225.26.25:7894'  # Internal IP with protocol/port
```

<ParamField path="actions.allowedDomains" type="array">
  Domains allowed for Agent Actions OpenAPI specs
  
  <Warning>
  If not configured, SSRF targets are blocked (localhost, private IPs, .internal/.local TLDs).
  To allow internal targets, you MUST explicitly add them.
  </Warning>
  
  Supports:
  - Wildcards: `*.example.com`
  - Protocol/port restrictions: `https://api.example.com:8443`
</ParamField>

## MCP Settings

```yaml
mcpSettings:
  allowedDomains:
    - 'host.docker.internal'
    - 'localhost'
    - '*.example.com'
    - 'https://secure.api.com'
```

<ParamField path="mcpSettings.allowedDomains" type="array">
  Domains allowed for MCP remote transports (SSE, WebSocket, HTTP)
  
  <Warning>
  Required for Docker setups to allow `host.docker.internal`
  </Warning>
</ParamField>

## MCP Servers

```yaml
mcpServers:
  everything:
    url: http://localhost:3001/sse
    timeout: 60000
  puppeteer:
    type: stdio
    command: npx
    args:
      - -y
      - "@modelcontextprotocol/server-puppeteer"
    timeout: 300000
  filesystem:
    command: npx
    args:
      - -y
      - "@modelcontextprotocol/server-filesystem"
      - /home/user/LibreChat/
    iconPath: /home/user/LibreChat/client/public/assets/logo.svg
```

<ParamField path="mcpServers.<name>.type" type="string">
  MCP server transport type: `sse`, `stdio`, `websocket`, or `http`
  
  <Note>Can be omitted, defaults to `sse`</Note>
</ParamField>

<ParamField path="mcpServers.<name>.url" type="string">
  Server URL (for SSE/WebSocket/HTTP transports)
</ParamField>

<ParamField path="mcpServers.<name>.command" type="string">
  Command to run (for stdio transport)
</ParamField>

<ParamField path="mcpServers.<name>.args" type="array">
  Command arguments (for stdio transport)
</ParamField>

<ParamField path="mcpServers.<name>.timeout" type="number" default="60000">
  Timeout in milliseconds
</ParamField>

<ParamField path="mcpServers.<name>.iconPath" type="string">
  Path to custom icon for the server
</ParamField>

## Memory Configuration

```yaml
memory:
  disabled: false
  validKeys: ["preferences", "work_info", "personal_info", "skills", "interests", "context"]
  tokenLimit: 10000
  personalize: true
  agent:
    id: "your-memory-agent-id"
    # OR define inline:
    # provider: "openai"
    # model: "gpt-4o-mini"
    # instructions: "You are a memory management assistant."
    # model_parameters:
    #   temperature: 0.1
```

<ParamField path="memory.disabled" type="boolean" default="false">
  Disable memory functionality
</ParamField>

<ParamField path="memory.validKeys" type="array">
  Restrict memory keys to specific values
</ParamField>

<ParamField path="memory.tokenLimit" type="number" default="10000">
  Maximum token limit for memory storage
</ParamField>

<ParamField path="memory.personalize" type="boolean" default="true">
  Enable personalization features (Personalization tab in settings)
</ParamField>

<ParamField path="memory.agent" type="object">
  Memory agent configuration - use existing agent by ID or define inline
</ParamField>
