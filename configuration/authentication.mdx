title: Authentication

LibreChat supports multiple authentication methods. Configure them using environment variables and `librechat.yaml`.

## Local Authentication

Email and password authentication is enabled by default.

<ParamField path="ALLOW_EMAIL_LOGIN" type="boolean" default="true">
  Enable email/password authentication
</ParamField>

<ParamField path="ALLOW_REGISTRATION" type="boolean" default="true">
  Allow new user registration
</ParamField>

<ParamField path="ALLOW_PASSWORD_RESET" type="boolean" default="false">
  Enable password reset functionality
</ParamField>

<ParamField path="ALLOW_UNVERIFIED_EMAIL_LOGIN" type="boolean" default="true">
  Allow login without email verification
</ParamField>

## Social Login (OAuth)

Enable social authentication providers.

<ParamField path="ALLOW_SOCIAL_LOGIN" type="boolean" default="false">
  Enable OAuth social login
</ParamField>

<ParamField path="ALLOW_SOCIAL_REGISTRATION" type="boolean" default="false">
  Allow registration via social providers
</ParamField>

### Configure Providers in YAML

```yaml
registration:
  socialLogins: ['github', 'google', 'discord', 'openid', 'facebook', 'apple', 'saml']
```

Generic OpenID Connect provider support.

</ParamField>

<ParamField path="OPENID_CLIENT_SECRET" type="string" required>
  OpenID Connect client secret
</ParamField>

<ParamField path="OPENID_ISSUER" type="string" required>
  OpenID Connect issuer URL
</ParamField>

<ParamField path="OPENID_SESSION_SECRET" type="string" required>

</ParamField>

<ParamField path="OPENID_SCOPE" type="string" default="openid profile email">
  OAuth scopes to request
</ParamField>

<ParamField path="OPENID_CALLBACK_URL" type="string" default="/oauth/openid/callback">

</ParamField>

### User Mapping

<ParamField path="OPENID_USERNAME_CLAIM" type="string">
  User info property for username
</ParamField>

<ParamField path="OPENID_NAME_CLAIM" type="string">
  User info property for display name
</ParamField>

<ParamField path="OPENID_EMAIL_CLAIM" type="string">
  User info claim for email/identifier
  
  <Note>Defaults to: email → preferred_username → upn</Note>
</ParamField>

### Role-Based Access

<ParamField path="OPENID_REQUIRED_ROLE" type="string">

</ParamField>

<ParamField path="OPENID_REQUIRED_ROLE_TOKEN_KIND" type="string">
  Token kind for role validation
</ParamField>

<ParamField path="OPENID_REQUIRED_ROLE_PARAMETER_PATH" type="string">
  Path to role in token
</ParamField>

<ParamField path="OPENID_ADMIN_ROLE" type="string">
  Role that grants admin access
</ParamField>

<ParamField path="OPENID_ADMIN_ROLE_PARAMETER_PATH" type="string">
  Path to admin role in token
</ParamField>

### UI Customization

<ParamField path="OPENID_BUTTON_LABEL" type="string">
  Custom label for OpenID login button
</ParamField>

<ParamField path="OPENID_IMAGE_URL" type="string">
  Custom icon URL for OpenID login button
</ParamField>

<ParamField path="OPENID_AUTO_REDIRECT" type="boolean" default="false">
  Automatically redirect to OpenID provider on login page
  
  <Warning>Only use if OpenID is your sole authentication method</Warning>
</ParamField>

### Advanced Options

<ParamField path="OPENID_USE_PKCE" type="boolean" default="false">
  Use PKCE (Proof Key for Code Exchange) for enhanced security
</ParamField>

<ParamField path="OPENID_REUSE_TOKENS" type="boolean">
  Reuse OpenID tokens for authentication instead of MongoDB session
</ParamField>

<ParamField path="OPENID_JWKS_URL_CACHE_ENABLED" type="boolean" default="true">
  Enable caching of JWKS signing key verification
</ParamField>

<ParamField path="OPENID_JWKS_URL_CACHE_TIME" type="number" default="600000">
  JWKS cache time in milliseconds (10 minutes)
</ParamField>

<ParamField path="OPENID_ON_BEHALF_FLOW_FOR_USERINFO_REQUIRED" type="boolean">
  Trigger token exchange flow for userinfo endpoint
</ParamField>

<ParamField path="OPENID_ON_BEHALF_FLOW_USERINFO_SCOPE" type="string" default="user.read">
  Scope for on-behalf flow (e.g., Microsoft Graph API)
</ParamField>

<ParamField path="OPENID_USE_END_SESSION_ENDPOINT" type="boolean">
  Use OpenID Connect end session endpoint for logout
</ParamField>

<ParamField path="OPENID_POST_LOGOUT_REDIRECT_URI" type="string">
  URL to redirect after OpenID logout
  
  <Note>Defaults to ${DOMAIN_CLIENT}/login</Note>
</ParamField>

<ParamField path="OPENID_AUDIENCE" type="string">
  Optional audience parameter for authorization requests
</ParamField>

<CodeGroup>
```bash .env
OPENID_CLIENT_ID=your-client-id
OPENID_CLIENT_SECRET=your-client-secret
OPENID_ISSUER=https://accounts.google.com
OPENID_SESSION_SECRET=your-session-secret
OPENID_SCOPE="openid profile email"
OPENID_CALLBACK_URL=/oauth/openid/callback
```

```yaml librechat.yaml
registration:
  socialLogins: ['openid']
```
</CodeGroup>

### UI Customization

  Custom label for SAML login button
</ParamField>

### Signature Settings

<ParamField path="SAML_USE_AUTHN_RESPONSE_SIGNED" type="boolean">
  Whether the SAML Response should be signed
  
  - `true`: Entire SAML Response will be signed
  - `false` or unset: Only SAML Assertion will be signed (default)
</ParamField>

```bash .env
SAML_ENTRY_POINT=https://idp.example.com/sso
SAML_ISSUER=librechat
SAML_CERT=MIIDXTCCAkWgAwIBAgIJAKL...
SAML_CALLBACK_URL=/oauth/saml/callback
SAML_SESSION_SECRET=your-session-secret

# Optional attribute mappings
SAML_EMAIL_CLAIM=email
SAML_USERNAME_CLAIM=username
SAML_GIVEN_NAME_CLAIM=firstName
SAML_FAMILY_NAME_CLAIM=lastName
```

</ParamField>

<ParamField path="LDAP_TLS_REJECT_UNAUTHORIZED" type="boolean">

</ParamField>

<ParamField path="LDAP_STARTTLS" type="boolean">
  Enable STARTTLS
</ParamField>

<ParamField path="LDAP_LOGIN_USES_USERNAME" type="boolean" default="false">
  Use username instead of email for login
</ParamField>

### Attribute Mappings

<ParamField path="LDAP_ID" type="string">
  LDAP attribute for user ID
</ParamField>

<ParamField path="LDAP_USERNAME" type="string">
  LDAP attribute for username
</ParamField>

<ParamField path="LDAP_EMAIL" type="string">
  LDAP attribute for email
</ParamField>

<ParamField path="LDAP_FULL_NAME" type="string">
  LDAP attribute for full name
</ParamField>

<CodeGroup>
```bash .env
LDAP_URL=ldap://ldap.example.com:389
LDAP_BIND_DN=cn=admin,dc=example,dc=com
LDAP_BIND_CREDENTIALS=admin-password
LDAP_USER_SEARCH_BASE=ou=users,dc=example,dc=com
LDAP_SEARCH_FILTER="mail="

# Set minimum password length to 1 for LDAP
MIN_PASSWORD_LENGTH=1
```
</CodeGroup>

<Note>
  Set `MIN_PASSWORD_LENGTH=1` when using LDAP to bypass local password validation, as LDAP servers handle their own password policies.
</Note>

## Two-Factor Authentication

Two-factor authentication (2FA) is available through the API endpoints:

- `POST /auth/2fa/enable` - Enable 2FA for user
- `POST /auth/2fa/verify` - Verify 2FA code
- `POST /auth/2fa/confirm` - Confirm 2FA setup
- `POST /auth/2fa/disable` - Disable 2FA
- `POST /auth/2fa/backup/regenerate` - Regenerate backup codes

No additional configuration required.
